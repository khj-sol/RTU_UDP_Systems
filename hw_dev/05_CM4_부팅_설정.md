# CM4 부팅 설정 (config.txt / Device Tree Overlay)

## 1. config.txt 위치

| OS 버전 | 경로 |
|---------|------|
| Bullseye 이전 | `/boot/config.txt` |
| Bookworm (2023~) | `/boot/firmware/config.txt` |
| Trixie (2025~) | `/boot/firmware/config.txt` |

## 2. UART 활성화 설정

### RTU 프로젝트 config.txt 설정
```ini
# ─── RS485 4CH UART (CM4-ETH-RS485-BASE-B) ───
# CH0: UART0 (GPIO 14/15) - 디버거
dtoverlay=uart0

# CH1: UART3 (GPIO 4/5) - 설비 통신
dtoverlay=uart3

# CH2: UART4 (GPIO 8/9) - DER-AVM
dtoverlay=uart4

# CH3: UART5 (GPIO 12/13) - 예비
dtoverlay=uart5

# Bluetooth 비활성화 (UART0 해제)
dtoverlay=disable-bt

# UART 하드웨어 활성화
enable_uart=1
```

### 오버레이 옵션
| 오버레이 | 활성화 UART | GPIO | 비고 |
|---------|------------|------|------|
| `dtoverlay=uart0` | UART0 → /dev/ttyAMA0 | GPIO 14/15 | |
| `dtoverlay=uart2` | UART2 → /dev/ttyAMA1 | GPIO 0/1 | I2C0과 충돌 |
| `dtoverlay=uart3` | UART3 → /dev/ttyAMA2 | GPIO 4/5 | |
| `dtoverlay=uart3,ctsrts` | +플로우 컨트롤 | +GPIO 6/7 | CTS/RTS 필요 시 |
| `dtoverlay=uart4` | UART4 → /dev/ttyAMA3 | GPIO 8/9 | SPI0과 충돌 |
| `dtoverlay=uart5` | UART5 → /dev/ttyAMA4 | GPIO 12/13 | PWM과 충돌 |
| `dtoverlay=disable-bt` | BT 해제 | - | UART0 GPIO 14/15 해제 |

### 미니 UART 보드레이트 안정화
미니 UART(UART1) 사용 시 CPU 클럭 고정 필요:
```ini
core_freq=250
core_freq_min=250
```
**참고**: PL011 UART (UART0,2,3,4,5)는 클럭 영향 없음 → RTU 프로젝트에서는 불필요

## 3. 시리얼 콘솔 비활성화

UART를 애플리케이션에서 사용하려면 리눅스 시리얼 콘솔을 해제해야 합니다.

### cmdline.txt 수정
```bash
# /boot/firmware/cmdline.txt에서 아래 문자열 제거
console=serial0,115200
# 또는
console=ttyAMA0,115200
```

### 시리얼 getty 서비스 비활성화
```bash
sudo systemctl stop serial-getty@ttyAMA0.service
sudo systemctl disable serial-getty@ttyAMA0.service
sudo systemctl disable serial-getty@ttyAMA3.service
sudo systemctl disable serial-getty@ttyAMA4.service
sudo systemctl disable serial-getty@ttyAMA5.service
```

### raspi-config 사용
```bash
sudo raspi-config
# → Interface Options → Serial Port
# → "login shell over serial?" → No
# → "serial port hardware enabled?" → Yes
```

## 4. Device Tree Overlay 디버깅

### 적용된 오버레이 확인
```bash
# 현재 로드된 오버레이
ls /proc/device-tree/soc/serial@*

# UART 디바이스 확인
ls -la /dev/ttyAMA*
ls -la /dev/ttyS*

# 디바이스 트리 정보
dtc -I fs /proc/device-tree 2>/dev/null | grep -A5 uart
```

### ttyAMA 번호 주의사항

**Bookworm 이후 알려진 문제**: UART를 여러 개 활성화하면 `/dev/ttyAMA*` 번호가
UART 하드웨어 번호와 일치하지 않을 수 있습니다.

```
# 예상 매핑
UART3 → /dev/ttyAMA2
UART4 → /dev/ttyAMA3

# 실제 매핑 (커널 버전에 따라 다를 수 있음)
UART3 → /dev/ttyAMA3  ← 번호 다름!
UART4 → /dev/ttyAMA4
```

확인 방법:
```bash
# 각 UART의 실제 디바이스 확인
cat /proc/tty/driver/ttyAMA
```

## 5. CM4 vs Pi 4 UART 차이

| 항목 | CM4 | Pi 4 Model B |
|------|-----|-------------|
| SoC | BCM2711 (동일) | BCM2711 (동일) |
| 사용 가능 UART | 6개 (캐리어 보드에 따라) | 2개 (40핀 헤더 제한) |
| UART 기본 상태 | 모두 비활성 | UART0/1만 활성 |
| GPIO 접근 | 200핀 메자닌 (전체 노출) | 40핀 헤더 (제한적) |
| SPI0/UART4 충돌 | 캐리어 보드 설계로 회피 가능 | 40핀 헤더에서 충돌 |
