# RS485 통신 규격

## 1. EIA/TIA-485 전기 규격

### 기본 특성
| 항목 | 값 |
|------|-----|
| 표준 | EIA/TIA-485 (1983년 제정) |
| 신호 방식 | 차동 신호 (Differential Signaling, A/B 2선) |
| 드라이버 출력 | 최소 1.5V 차동 (54옴 부하) |
| 수신 감도 | 최소 200mV 차동 |
| 공통 모드 전압 | -7V ~ +12V |
| 최대 케이블 길이 | 1,200m (9,600bps 기준) |
| 최대 장비 수 | 32대 (표준) / 128대 (1/4 유닛로드) / 256대 (1/8 유닛로드) |

### 통신 모드
| 모드 | 배선 | 설명 |
|------|------|------|
| Half-duplex (2선) | A, B | TX/RX 공유, 한 번에 1대만 송신. **Modbus RTU 표준** |
| Full-duplex (4선) | TX+/TX-, RX+/RX- | 동시 송수신, RS-422 호환 |

**RTU 프로젝트**: Half-duplex (2선) 사용

### 종단 저항 (Termination)
```
┌────────┐                                    ┌────────┐
│ Master ├──┬─── A ─────────────────── A ──┬──┤ Slave  │
│        │  │                              │  │        │
│        │ [120Ω]                         [120Ω]       │
│        │  │                              │  │        │
│        ├──┴─── B ─────────────────── B ──┴──┤        │
└────────┘                                    └────────┘
```

- 양 끝단에 **120옴 종단 저항** 각 1개씩 (총 2개)
- 중간 장비에는 종단 저항 설치하지 않음
- Fail-safe 바이어스: A 풀업, B 풀다운 권장 (버스 무활동 시 안정)

## 2. Modbus RTU 프레임 구조

### 프레임 형식
```
┌──────────┬──────────┬──────────┬──────────┐
│ Slave    │ Function │ Data     │ CRC-16   │
│ Address  │ Code     │ (가변)   │          │
│ 1 byte   │ 1 byte   │ 0~252    │ 2 bytes  │
│          │          │ bytes    │ (Low,Hi) │
└──────────┴──────────┴──────────┴──────────┘
```

| 필드 | 크기 | 설명 |
|------|------|------|
| Slave Address | 1 byte | 1~247 (0 = 브로드캐스트) |
| Function Code | 1 byte | 명령 코드 |
| Data | 0~252 bytes | 레지스터 주소, 개수, 값 등 |
| CRC-16 | 2 bytes | Low byte 먼저, 다항식 0xA001 |

최대 프레임 크기: **256 bytes**

### 프레임 타이밍
| 항목 | 값 |
|------|-----|
| 프레임 간 최소 묵음 | 3.5 문자 시간 |
| 프레임 내 최대 갭 | 1.5 문자 시간 (초과 시 프레임 폐기) |

### 시리얼 워드 형식
- 11비트: 시작(1) + 데이터(8) + 패리티(1) + 정지(1)
- 패리티 없을 때: 시작(1) + 데이터(8) + 정지(2)

### 주요 Function Code

| FC | 코드 | 설명 | 용도 |
|----|------|------|------|
| FC03 | 0x03 | Read Holding Registers | 인버터 상태/전력 데이터 읽기 |
| FC04 | 0x04 | Read Input Registers | 계측 데이터 읽기 |
| FC06 | 0x06 | Write Single Register | 단일 제어 파라미터 쓰기 |
| FC16 | 0x10 | Write Multiple Registers | 복수 레지스터 일괄 쓰기 |

## 4. 배선 권장사항

| 항목 | 권장 |
|------|------|
| 케이블 | 차폐 트위스트 페어 (STP) |
| 단면적 | 0.5mm² 이상 (AWG 22~24) |
| 최대 길이 | 1,200m (9600bps) |
| 종단 저항 | 양 끝단 120옴 |
| 접지 | 차폐선은 한쪽만 접지 (그라운드 루프 방지) |
| 장비 간격 | 최소 1m 이상 스터브 금지 (데이지 체인 구성) |

### 올바른 배선 (데이지 체인)
```
[Master]──[Dev1]──[Dev2]──[Dev3]──[Dev4]
  120Ω                                120Ω
```

### 잘못된 배선 (스타 토폴로지) - 사용 금지
```
         [Dev1]
          │
[Master]──┼──[Dev2]    ← 반사파 발생, 통신 에러
          │
         [Dev3]
```
