# 테스트 서버 사용 매뉴얼

## 1. 개요

RTU 개발 및 디버깅용 대화형 UDP 서버입니다.
H03 제어 명령을 수동으로 전송하고, 수신 패킷을 실시간으로 확인할 수 있습니다.

---

## 2. 실행 방법

### 배치파일 실행
```
launchers\START_TEST_SERVER.bat
```

### 직접 실행
```bash
python pc_programs/udp_test_server.py
python pc_programs/udp_test_server.py --port 13132 --rtu-port 9100
python pc_programs/udp_test_server.py --headless
```

### 커맨드라인 옵션
| 옵션 | 기본값 | 설명 |
|------|--------|------|
| `--port` | 13132 | UDP 수신 포트 |
| `--rtu-port` | 9100 | RTU 통신 포트 |
| `--ftp-port` | 21 | FTP 서버 포트 |
| `--ftp-root` | ./firmware | FTP 루트 디렉토리 |
| `--headless` | 꺼짐 | 메뉴 없이 수신만 |

---

## 3. 대화형 메뉴

서버 시작 후 아래 메뉴가 표시됩니다:

### 제어 명령 (H03)
| 번호 | 명령 | 값 범위 |
|------|------|---------|
| 1 | 인버터 ON/OFF | 0=ON, 1=OFF |
| 2 | 유효전력 제한 | 0~1000 (0~100%) |
| 3 | 역률 | -1.0 ~ 1.0 |
| 4 | 무효전력 | -100% ~ 100% |
| 5 | 제어 초기화 | - |
| 6 | 제어 상태 확인 | - |
| 7 | 모델 정보 조회 | - |
| 8 | IV 스캔 | - |
| 9 | RTU 정보 조회 | - |
| 10 | RTU 재부팅 | - |

### 추가 기능
| 키 | 기능 |
|----|------|
| U | 펌웨어 업데이트 (H07 전송) |
| F | FTP 서버 제어 (시작/중지/사용자관리) |
| S | 통계 표시 (패킷별 송수신 건수) |
| Q | 종료 |

---

## 4. 패킷 표시

### H01 수신 시
```
[10:15:30] H01 Periodic Data from RTU 12345678
  Device: Inverter #1 (Solarize)
  PV Power: 45.2 kW | AC Power: 44.8 kW
  Status: ON_GRID | PF: 0.995
```

### H05 이벤트 표시
| Body Type | 이벤트 |
|-----------|--------|
| 1 | 하트비트 PING |
| 2 | RTU 이벤트 (문자열) |
| 3 | RTU 정보 (모델, 시리얼, 펌웨어) |
| 4 | 정전 발생 |
| 5 | 정전 복구 |
| 6 | 인버터 모델 정보 |
| 10 | 제어 상태 확인 결과 |
| 11 | 제어 실행 결과 |
| 12 | IV 스캔 완료 |
| 15 | IV 스캔 데이터 (스트링별) |

### 디버깅 기능
- 타임스탬프 검증 (서버 시간과 60초 이상 차이 시 경고)
- 백업 데이터 표시 ("BACKUP" 플래그 + 경과 시간)
- 통신 실패 감지 (body_type = -1)

---

## 5. FTP 서버 서브메뉴 ([F])

| 기능 | 설명 |
|------|------|
| FTP 시작 | pyftpdlib 기반 FTP 서버 시작 |
| FTP 중지 | FTP 서버 종료 |
| 사용자 추가 | FTP 로그인 계정 추가 |
| 사용자 삭제 | FTP 계정 삭제 |
| 폴더 열기 | firmware 디렉토리를 탐색기에서 열기 |
| 상태 표시 | FTP 서버 상태 및 로컬 IP 표시 |

## 6. 사용 시나리오

### RTU 연결 테스트
1. 테스트 서버 실행
2. RTU 시작 (`-s` 시뮬레이션 모드)
3. H05 최초 연결 수신 확인
4. 60초 후 H01 주기 데이터 수신 확인

### 제어 명령 테스트
1. H01 데이터 수신 확인 (RTU 연결됨)
2. 메뉴에서 `1` 선택 (인버터 ON/OFF)
3. 장비 번호, 값 입력
4. H04 응답 수신 확인
5. H05 제어 결과 수신 확인

### IV 스캔 테스트
1. 메뉴에서 `8` 선택 (IV 스캔)
2. H05(12) IV 스캔 완료 대기 (5~30초)
3. H05(15) 스트링별 IV 데이터 수신 확인
