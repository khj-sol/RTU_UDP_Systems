# Model Maker 사용 매뉴얼

## 1. 개요

새로운 인버터 제조사의 Modbus 프로토콜 PDF를 읽어 RTU 호환 레지스터맵 Python 파일을
자동 생성하는 GUI 도구입니다. 3단계 파이프라인(PDF → Excel → 매핑 → 코드)으로 동작합니다.

---

## 2. 실행 방법

### 배치파일 실행
```
launchers\START_MODEL_MAKER.bat
```
- PyMuPDF, openpyxl, anthropic 자동 설치

### 직접 실행
```bash
python -m model_maker.modbus_to_udp_mapper
```

---

## 3. GUI 구성 (3개 탭)

### Tab 1: Stage 1 — PDF → Excel

인버터 Modbus 프로토콜 PDF에서 레지스터 정보를 추출합니다.

#### 사용 절차
1. **Open PDF** 클릭 → 제조사 Modbus 프로토콜 PDF 선택
2. 모드 선택:
   - **Offline**: PyMuPDF + 내장 레퍼런스 라이브러리 (AI 불필요)
   - **AI**: Claude API로 모호한 레지스터 해석 (정확도 높음)
3. **자동생성** 또는 **AI 생성** 클릭
4. 결과: 9개 시트가 포함된 Excel 파일 생성

#### 생성되는 9개 시트
| 시트 | 내용 |
|------|------|
| Registers | 전체 레지스터 목록 (주소, 이름, 타입, 스케일) |
| Status_Definitions | 인버터 상태 코드 정의 |
| Error_Fault_Codes | 오류/고장 코드 |
| Grid_Codes | 계통 연계 코드 |
| Fuse_Open_Data | 퓨즈 개방 데이터 |
| Status_Bits_03FC | 상태 비트 정의 |
| Fuse_Check_Mask | 퓨즈 체크 마스크 |
| REMS_Error_Bits | REMS 에러 비트 |
| REMS_Registers | REMS 레지스터 |

#### 편집 기능
- 각 시트의 셀을 직접 편집 가능 (Definition, Data_Type, Unit, Scale_Factor 등)
- **Save Excel**: 편집 내용 Excel로 저장
- **Open in Excel**: 시스템 Excel에서 열기

---

### Tab 2: Stage 2 — UDP 매핑

Stage 1 Excel의 레지스터를 Solarize UDP 프로토콜 필드에 매핑합니다.

#### 사용 절차
1. Stage 1 Excel 경로 확인 (Tab1에서 자동 연결)
2. **UDP 매핑 실행 (Offline)** 또는 **(AI)** 클릭
3. 결과: 4개 시트의 매핑 Excel 생성

#### 생성되는 4개 시트
| 시트 | 내용 | 색상 |
|------|------|------|
| Register_Mapping | 장비 레지스터 → UDP 필드 매핑 | 녹색=자동매칭, 적색=미매핑 |
| Status_Mapping | 상태/고장 코드 → InverterMode 매핑 | |
| DER_AVM_Control | H03 제어 요청 매핑 | |
| DEA_AVM_Monitor | 제어 결과 모니터링 매핑 | |

---

### Tab 3: Stage 3 — 코드 생성

매핑 Excel에서 Python 레지스터맵 코드를 생성합니다.

#### 설정 항목
| 항목 | 설명 | 필수 |
|------|------|------|
| Manufacturer | 제조사명 | O |
| Protocol Name | 프로토콜 식별자 (예: newbrand) | O |
| Model # | 모델 번호 | O |
| Class Name | **반드시 `RegisterMap`** | O |
| Function Code | FC03 (Holding) / FC04 (Input) | O |
| MPPT count | MPPT 트래커 수 (1~8) | O |
| String count | PV 스트링 수 (1~24) | O |
| IV Scan | IV 커브 스캔 지원 여부 | |
| DER-AVM | DER-AVM 제어 필드 포함 여부 | |

#### 사용 절차
1. 설정 항목 입력
2. **레지스터 생성** 클릭 → 코드 생성 + 자동 검증
3. Code Preview에서 생성 코드 확인
4. Validation Results에서 PASS/WARN/FAIL 확인
5. **Save to common/** 클릭 → `common/{protocol}_registers.py` 저장
6. Config Editor에서 `rs485_ch1.ini`에 장비 추가 (Add Model)

#### 검증 항목
| 상태 | 설명 |
|------|------|
| PASS | 필수 요소 모두 존재 |
| WARN | 권장 사항 누락 (동작에 영향 없음) |
| FAIL | 필수 요소 누락 (RTU 호환 불가) |

## 4. 생성 코드 필수 요소

RTU와 호환되려면 다음 요소가 반드시 포함되어야 합니다:

```python
class RegisterMap:
    """레지스터 주소 상수"""
    PV1_VOLTAGE = 0x0100
    PV1_CURRENT = 0x0101
    # ...

class InverterMode:
    INITIAL = 0
    STANDBY = 1
    ON_GRID = 2
    FAULT = 3
    SHUTDOWN = 4

    @staticmethod
    def to_string(value): ...

SCALE = {
    'voltage': 0.1,
    'current': 0.01,
    'power': 0.1,
    'frequency': 0.01,
    'power_factor': 0.001,
}

def registers_to_u32(high, low): ...
def registers_to_s32(high, low): ...
```

## 5. 레지스터 파일 관리

### 좌측 패널: Register Files
- `common/` 디렉토리의 `*_registers.py` 파일 목록
- Open: 코드 미리보기에 로드
- Delete: 파일 삭제 (device_models.ini, rs485_ch1.ini에서도 제거)
- Refresh: 목록 새로고침

### 보호 파일
아래 파일은 Model Maker로 덮어쓸 수 없습니다:
- `solarize_registers.py`
- `kstar_registers.py`
- `huawei_registers.py`

테스트 시 별도 프로토콜명 사용 (예: `newbrand_test`)

## 6. 펌웨어 패키지 빌드

Tab 3의 **Build Package** 버튼으로 RTU 배포용 패키지를 생성합니다.

### 포함 파일
| 디렉토리 | 확장자 | 재귀 |
|----------|--------|------|
| rtu_program/ | .py | O (하위 포함) |
| common/ | .py | X (최상위만) |
| config/ | .ini | X (최상위만) |

### 출력
- 위치: `pc_programs/firmware/rtu_firmware_YYYYMMDD_HHMMSS.tar.gz`
- 배포: 대시보드 Firmware 탭 → RTU에 업로드

## 7. AI 설정

### API 키 설정
Tab 1의 **API Settings** 버튼으로 Claude API 키를 설정합니다.
- API 키는 로컬에 저장됨
- AI 모드 사용 시에만 필요

### AI 모드 vs Offline 모드
| 항목 | Offline | AI |
|------|---------|-----|
| 속도 | 빠름 (수초) | 느림 (수십초) |
| 정확도 | 레퍼런스 기반 | 문맥 이해 |
| 비용 | 무료 | API 비용 발생 |
| 인터넷 | 불필요 | 필요 |
| 적합 | Solarize 호환 인버터 | 새로운/비표준 인버터 |
