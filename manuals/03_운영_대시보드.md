# 운영 대시보드 사용 매뉴얼

## 1. 개요

FastAPI 기반 웹 대시보드로, UDP 서버 + 웹 UI + WebSocket 실시간 업데이트를 통합합니다.
RTU로부터 데이터를 수신하고, 제어 명령을 전송하며, 펌웨어 배포를 관리합니다.

---

## 2. 실행 방법

### 배치파일 실행 (Windows)
```
launchers\START_DASHBOARD.bat
```
- Watchdog 자동 재시작 포함
- 비정상 종료 시 5초 후 재실행

### 직접 실행
```bash
python -m web_server_prod.main
```

### 환경변수 설정 (선택)
| 환경변수 | 기본값 | 설명 |
|----------|--------|------|
| `RTU_UDP_PORT` | 13132 | UDP 수신 포트 |
| `RTU_WEB_PORT` | 8080 | 웹 서버 포트 |
| `RTU_DB_PATH` | `web_server_prod/rtu_dashboard.db` | SQLite DB 경로 |
| `RTU_FTP_USER` | rtu | FTP 사용자명 |
| `RTU_FTP_PASS` | 1234 | FTP 비밀번호 |
| `RTU_SSH_USER` | pi | RTU SSH 사용자 |
| `RTU_SSH_PASS` | raspberry | RTU SSH 비밀번호 |

## 4. REST API

### RTU 관리
| Method | Endpoint | 설명 |
|--------|----------|------|
| GET | `/api/rtus` | 모든 RTU 목록 |
| GET | `/api/rtus/{rtu_id}` | RTU 상세 정보 |
| DELETE | `/api/rtus/{rtu_id}` | RTU 삭제 |
| GET | `/api/rtus/{rtu_id}/devices` | RTU 장비 데이터 |

### 데이터 조회
| Method | Endpoint | 파라미터 | 설명 |
|--------|----------|----------|------|
| GET | `/api/data/inverter` | rtu_id, device_num, from_ts, to_ts, limit | 인버터 이력 |
| GET | `/api/data/relay` | 동일 | 릴레이 이력 |
| GET | `/api/rtus/{rtu_id}/iv_scan` | - | IV 스캔 데이터 |

### 제어 명령
| Method | Endpoint | Body | 설명 |
|--------|----------|------|------|
| POST | `/api/control/on_off` | `{rtu_id, device_num, value}` | 인버터 ON/OFF |
| POST | `/api/control/active_power` | 동일 (value: 0~1000) | 유효전력 제한 |
| POST | `/api/control/power_factor` | 동일 (value: -1000~1000) | 역률 |
| POST | `/api/control/reactive_power` | 동일 (value: -484~484) | 무효전력 |
| POST | `/api/control/init` | `{rtu_id, device_num}` | 제어 초기화 |
| POST | `/api/control/check` | 동일 | 제어 상태 확인 |
| POST | `/api/control/iv_scan` | 동일 | IV 스캔 실행 |
| POST | `/api/control/reboot` | `{rtu_id}` | RTU 재부팅 |

### 펌웨어 관리
| Method | Endpoint | 설명 |
|--------|----------|------|
| GET | `/api/firmware/list` | 펌웨어 목록 (.tar.gz) |
| POST | `/api/firmware/upload` | 펌웨어 업로드 (max 500MB) |
| POST | `/api/firmware/update` | RTU에 펌웨어 업데이트 전송 |

### 설정 관리
| Method | Endpoint | 설명 |
|--------|----------|------|
| GET | `/api/config/files` | 설정 파일 목록 |
| GET | `/api/config/read?path=...` | 설정 파일 읽기 |
| POST | `/api/config/write` | 설정 파일 쓰기 |

## 6. 내장 기능

### FTP 서버
- 포트: 21
- 루트: `pc_programs/firmware/`
- RTU 펌웨어 다운로드용
- pyftpdlib 필요

### 백그라운드 작업
| 작업 | 주기 | 설명 |
|------|------|------|
| 비활성 RTU 감지 | 60초 | 120~660초 무응답 시 오프라인 처리 |
| 데이터 보존 정리 | 1시간 | 30일 이상 데이터 삭제 |
| WAL 체크포인트 | 1시간 | SQLite WAL 파일 정리 |
| 다운샘플링 | 매일 03:00 | 30일+ 데이터를 5분/1시간 평균으로 축약 |
| 주기적 상태 확인 | 30초 | 모든 인버터에 H03 상태 확인 전송 |

### 보안
- 환경변수 기반 자격증명
- SFTP 경로 화이트리스트 (`/home/pi/config`, `/home/pi/common`)
- 펌웨어 파일명 검증 (.tar.gz만 허용, 경로 탐색 차단)
- 중복 패킷 감지 및 속도 제한
