# 장비 시뮬레이터 사용 매뉴얼

## 1. 개요

PC에서 실제 인버터/릴레이/기상센서 없이 Modbus RTU 슬레이브를 시뮬레이션합니다.
하나의 COM 포트에서 여러 슬레이브 ID로 다중 장비를 에뮬레이션합니다.

---

## 2. 실행 방법

### 배치파일 실행
```
launchers\START_SIMULATOR.bat
```

### 직접 실행
```bash
python pc_programs/equipment_simulator.py
python pc_programs/equipment_simulator.py --port COM10 --baudrate 9600
python pc_programs/equipment_simulator.py --config my_config.json
python pc_programs/equipment_simulator.py --headless
```

### 커맨드라인 옵션
| 옵션 | 기본값 | 설명 |
|------|--------|------|
| `--port` | COM10 (자동감지) | 시리얼 포트 |
| `--baudrate` | 9600 | 통신 속도 |
| `--config` | simulator_config.json | 설정 파일 경로 |
| `--headless` | 꺼짐 | 상태 표시 없이 실행 |

### 필수 요건
- **USB-RS485 어댑터** 1개 (시뮬레이터용)
- **pymodbus** 라이브러리 (`pip install pymodbus`)
- RTU와 연결하려면 **USB-RS485 어댑터 2개** 필요 (시뮬레이터 + RTU 각각)

## 3. 설정 파일 (simulator_config.json)

```json
{
  "port": "COM10",
  "baudrate": 9600,
  "devices": [
    {
      "slave_id": 1,
      "type": "inverter",
      "name": "Solarize Verterking",
      "protocol": "solarize",
      "model_id": 1
    },
    {
      "slave_id": 2,
      "type": "inverter",
      "name": "Huawei SUN2000",
      "protocol": "huawei",
      "model_id": 2
    },
    {
      "slave_id": 3,
      "type": "inverter",
      "name": "Kstar KSG-60KT",
      "protocol": "kstar",
      "model_id": 3
    },
    {
      "slave_id": 4,
      "type": "relay",
      "name": "KDU-300",
      "protocol": "relay",
      "model_id": 1
    },
    {
      "slave_id": 5,
      "type": "weather",
      "name": "SEM5046",
      "protocol": "weather",
      "model_id": 1
    }
  ]
}
```

### 설정 필드
| 필드 | 타입 | 설명 |
|------|------|------|
| slave_id | int | Modbus 슬레이브 주소 (1~247) |
| type | string | inverter / relay / weather |
| name | string | 표시 이름 |
| protocol | string | solarize / huawei / kstar / relay / weather |
| model_id | int | 모델 ID (device_models.ini 참조) |

## 4. 시뮬레이션 데이터

### 인버터 (Solarize)
- **시간 기반 일사량**: 6AM 일출 → 12PM 피크(1000 W/m²) → 6PM 일몰
- **PV 전압**: 200~450V (일사량 비례)
- **PV 전류**: 0~10A (일사량 비례)
- **AC 전압**: 380V ±10V (3상)
- **AC 전력**: PV 전력 × 효율 (약 97%)
- **MPPT**: 4채널, 각각 독립 전압/전류
- **스트링**: 8개, 전류에 ±5% 편차
- **IV 스캔**: 64포인트 I-V 커브 (5초 소요)
- **상태**: ON_GRID, STANDBY, FAULT, SHUTDOWN

### 보호 릴레이 (KDU-300)
- **3상 전압**: R/S/T 220V ±5V
- **3상 전류**: 0~100A
- **전력**: 3상 합산
- **역률**: 0.9~1.0
- **주파수**: 60Hz
- **DO/DI**: 디지털 출력/입력 상태

### 기상센서 (SEM5046)
- **일사량**: 0~1000 W/m² (시간 패턴 + 구름 변동)
- **기온**: 15~35°C (일중 변화)
- **모듈온도**: 기온 + 5~10°C
- **습도**: 40~90% RH
- **풍속**: 0~10 m/s
- **풍향**: 0~360°
- **기압**: 1010~1020 hPa

## 5. 대화형 메뉴

| 키 | 기능 |
|----|------|
| R | 설정 파일 다시 로드 |
| C | 로그 지우기 |
| Q | 종료 |

### 실시간 표시
- 각 장비의 현재 데이터 (2초 갱신)
- Modbus 트랜잭션 로그 (최근 100건)
- 제어 레지스터 상태
- IV 스캔 진행 상태

## 6. PC 시연 구성

### 필요 장비
- USB-RS485 어댑터 2개 (또는 RS485 루프백 케이블)
- PC 1대

### 연결
```
[시뮬레이터 PC]              [RTU PC 또는 동일 PC]
  COM10 (USB-RS485) ←→ RS485 Bus ←→ COM3 (USB-RS485)
  equipment_simulator.py           rtu_client.py -c config
```

### 설정
1. `simulator_config.json`에서 `port`를 시뮬레이터 COM 포트로 설정
2. `config/rtu_config.ini`에서 `serial_port`를 RTU COM 포트로 설정
3. `config/rtu_config.ini`에서 `mode = serial` 설정
4. 시뮬레이터 먼저 실행 → RTU 실행 → 테스트 서버 실행

## 7. 문제 해결

| 증상 | 원인 | 해결 |
|------|------|------|
| "Could not start listen" | COM 포트 점유 | 다른 프로그램 종료, 포트 변경 |
| 데이터 안 바뀜 | 시간대 문제 | 시뮬레이터 로그에서 일사량 패턴 확인 |
| Modbus 응답 없음 | slave_id 불일치 | rs485_ch1.ini와 simulator_config.json 비교 |
| pymodbus 에러 | 버전 불일치 | `pip install pymodbus>=3.7` |
